## Symptom

Operator testing the 1stcontact builder chat repeatedly hits a "Browser
Rendering budget exhausted" stop after a few page renders, blocking iterative
testing. No budget was ever requested by the operator.

Two operator asks:
1. A session-reset button at the top of the chat (blank page, new session).
2. The "session budget" raised so high it is never hit during testing.

## Root cause (investigation — corrected)

- The browser-render session budget default WAS already raised to 1e9s
  ("effectively infinite") in source by commit ed5168a3 (BUG-17). There is NO
  low override anywhere in source; callers use the default.
- BUT ed5168a3 is on `xgd-working` and has NOT reached `main`, so it was never
  reconciled/deployed. The running builder serves a bundle compiled before the
  fix → still enforces the old 50s session / 200s day caps.
- The builder runs as a long-lived `wrangler dev` (apps/control-app `pnpm dev`).
  Its bundler watches the app's own src, not the workspace dependency
  `packages/web-fetch-safety`, so the live dev server kept serving the stale
  50s bundle even after the source fix landed.
- The on-disk `dist/browser-budget.js` was also stale (showed 50); rebuilt to
  1e9. `dist` is gitignored and not the resolution path (package resolves to
  src/index.ts), so it was a red herring, now cleared.

## Fix

1. (Feature) Add a "New session" control to the top of the builder chat panel
   (`packages/builder-ui/src/components/chat-panel.ts`), wired through `main.ts`
   to create a fresh chat session via `createSession` and reset the in-view
   history to a blank page. As a side effect this mints a new `sessionId`, which
   resets the per-session browser-budget KV counter — so it also unblocks a
   session that has exhausted its render budget.
2. (Budget) No source change required — source default is already 1e9. The
   operational fix is to restart `pnpm dev` (local) so the worker rebundles from
   current src, and to land ed5168a3 on `main` + redeploy for production.

## Test plan

  no longer the active one.
