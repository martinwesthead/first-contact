# BUG-7 regression test fails: build-builder-bundle.mjs emits combined `{builder,app}.js` message

## Symptom

`tests/test_UAT_FC_BUG-7_build_bundle_watch.test.ts` → "one-shot mode (no --watch) builds the bundle and exits 0" fails:

```
AssertionError: expected '\n  apps/control-app/public/_assets/a…' to match /Built .*builder\.js/
  expect(result.stdout).toMatch(/Built .*builder\.js/);   // line 78
```

The build succeeds (exit 0, bundle written) — only the stdout assertion fails. The companion watch-mode test in the same file asserts a `Watching` marker and is unaffected.

## Root cause

REQ-17 (app shell, request-7af3c503) extended `apps/control-app/scripts/build-builder-bundle.mjs` to bundle **two** entries — `builder.js` (the standalone `/builder` SPA) and `app.js` (the `/app` shell). Its console output was collapsed into a single brace-style line:

- line 50: `console.log(\`Watching builder-ui entries → ${outDir}/{builder,app}.js\`)`
- line 53: `console.log(\`Built ${outDir}/{builder,app}.js\`)`

The BUG-7 one-shot assertion `/Built .*builder\.js/` no longer matches: the literal output is `…/{builder,app}.js`, where `builder` is followed by `,app}.js`, not `.js`. The test was written when the script built only `builder.js` and was not updated when REQ-17 added the second entry.

This is a pre-existing failure on `xgd-working` (the build-script change is already committed); it is **not** introduced by REQ-16. Surfaced while validating REQ-16 (the Assets tab), which is independent of the build-message format.

## Fix options (pick one)

1. **Update the test assertion** to match the combined output, e.g. `expect(result.stdout).toMatch(/Built .*\{builder,app\}\.js/)` (or assert on the bundle file existing + non-zero size, which the test already does, and drop the brittle stdout-format check).
2. **Emit one log line per built bundle** in the script (`Built …/builder.js`, `Built …/app.js`), which keeps the existing assertion valid and is friendlier for future entries.

Option 2 is preferred — per-entry log lines are more robust to further entry additions and keep both standalone and shell builds independently greppable.

## Acceptance

- The BUG-7 one-shot and watch tests both pass.
- The build still produces `builder.js` and `app.js` (exit 0, non-zero sizes).
- No change to bundle contents — log/test only.
