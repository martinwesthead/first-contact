## Symptom

`apps/control-app` `pnpm dev` runs `wrangler dev` for the worker. wrangler
re-bundles when the worker's own entry (`src/index.ts`) changes, but it does
NOT watch the source of linked workspace packages (`packages/*`). So an edit to
a shared package (e.g. `@gendev/web-fetch-safety`) never reaches the running
worker until the dev server is fully restarted. This caused BUG-19's symptom:
the browser-budget fix (1e9s) sat in source for days while the live worker kept
serving the stale 50s bundle.

## Root cause

wrangler's file watcher covers the entry's direct file but not transitive
sources resolved through pnpm symlinks (`node_modules/@gendev/* -> ../../packages/*`).

## Fix

Add a dev-only watcher (`apps/control-app/scripts/dev-worker-rebuild.mjs`) wired
into `pnpm dev` alongside `wrangler dev`. It watches the `src/` of every
`workspace:*` dependency of the app and, on a source change, bumps the mtime of
the worker entry (`src/index.ts`). That nudges wrangler's existing rebuild path
to re-bundle the full graph — picking up the workspace change — without
restarting the process (port + miniflare state preserved).

Exposed, testable helpers:
- workspaceDepSrcDirs(appRoot) — resolves each workspace dep to its real src dir
- isWatchableSourceFile(name) — source-file extension filter
- touchEntry(path) — bump mtime to trigger a re-bundle

## Test plan

- test_UAT_FC_<ID>_* : workspaceDepSrcDirs returns all workspace deps' src dirs
  for control-app; isWatchableSourceFile accepts ts/mjs/json and rejects others;
  touchEntry bumps a file's mtime; the `dev` script wires in the watcher.

## Notes

- The front-end builder bundle is rebuilt by a separate watcher
  (`build:bundle:watch`); this ticket only addresses the worker bundle.
