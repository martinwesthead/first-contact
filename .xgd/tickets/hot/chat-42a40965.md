Before designing the panels themselves, two foundational questions shape everything downstream:

**1. Where does this UI live?** DOC-5 calls for a separate "GenDev Labs control application frontend" hosted on Cloudflare Workers Static Assets — a different surface from the XGD operator dashboard. So we'd be building a new app (likely modern JS/TS targeting Workers), reusing the *visual pattern* from the XGD chat panel but not the HTML/JS itself. Alternative: prototype inside the XGD dashboard's single-file architecture for speed, port later. The first commits us to the target architecture; the second is faster to iterate but throwaway.

**2. What does the right panel actually render?** This is the load-bearing decision. Three credible options:
- **Iframe of a preview build** — invoke the static renderer (D1 site definition + R2 assets → HTML), serve from a preview URL, iframe it. Truest "see what will ship", but every change needs a build round-trip. Durable Objects for "live preview sessions" (DOC-5) anticipate exactly this.
- **In-browser render of the site definition** — ship the shared component library to the builder app, render the site definition JSON directly in the right panel. Instant feedback, but the renderer has to be portable enough to run in both the builder UI and the static generator.
- **Hybrid** — in-browser render for structural changes (reorder sections, swap content), iframe rebuild on demand for theme/layout/SEO checks.

The chat-side commitment from DOC-4/5 is also non-trivial: the chat must produce *structured changes* (theme tokens, section reorder, content blocks, predefined components) — not arbitrary HTML. So this isn't just "wire up a chat panel"; the chat has to be backed by an AI orchestration layer that emits site-definition diffs the renderer can consume.

My recommendation: start by deciding **(2)** first — render strategy drives the data flow between chat and preview. If you want, I can sketch what a single round trip looks like (user message → AI → site-definition diff → preview update) under each option so we can pick. Which feels most worth exploring?
