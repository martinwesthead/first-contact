## Story
As a site builder, I want a testimonials module that presents either one featured client quote or a grid of client quotes, so that I can showcase customer trust on a small-business site without writing custom code.

## Description
Provides a content module registered in the framework module catalog at id `testimonials`, version 1. It offers two presentation variants:
- `single` — renders a single featured quote (the first supplied item only), regardless of how many items are provided.
- `grid` — renders a static, responsive card grid with one card per supplied item. A carousel/rotating presentation is explicitly out of scope for v1; the grid is pure CSS.

Each testimonial item carries a markdown `quote`, an attribution `name`, an optional `title`, and an optional `avatar` image. An optional section `heading` may precede the quotes. Configuration dials cover vertical spacing (`spacingTop` / `spacingBottom`), `surface` treatment, and text `align` (defaulting to centered, per testimonial convention).

In scope: registration in the catalog, the two variants, the alignment dial, optional avatar rendering, and markdown quote rendering. Out of scope: any rotating/carousel layout, and spacing/surface dial behavior shared with other modules (covered by the catalog-wide dial conventions).

## Technical Context
- Part of CAP-34 (Framework Module Catalog); sits alongside the other content modules (text-block, services-grid, image-gallery, etc.).
- The quote is rendered as already-parsed markdown HTML, matching the `set:html` convention used by text-block and services-grid; quote content is therefore not re-escaped.
- The avatar is rendered as a ~64px circular image using the asset reference's source and alt text.
- **Intent/code divergence (flag for regression):** REQ-40's implementation decisions (2026-06-20) specify per-variant item counts — `single` has no count enforcement and `grid` expects a minimum of 2 and maximum of 9 items. The shipped content schema instead enforces a single global bound of min 1 / max 9 across both variants; the per-variant grid minimum of 2 is not separately enforced at the schema level. This is documented here rather than asserted as enforced grid-min behavior in an acceptance criterion.
