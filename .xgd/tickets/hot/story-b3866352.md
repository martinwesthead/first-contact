## Story
**As an** operator building a site in 1st Contact, **I want** to paste the URL of an
existing website and have it converted into an editable draft of my own site,
**so that** I can reproduce a recognizable copy of that site within the framework
instead of building it from scratch.

## Description
This story covers the **mechanical convert-flow orchestration** that turns a
previously-analyzed Reference Digest into a draft the operator can iterate on.

In scope:
- The operator-triggered convert action and its companion confirmation action.
- A destructive-overwrite confirmation gate: the first attempt to convert a URL
  returns a confirmation request (no draft mutation); conversion proceeds only
  after the operator records consent. Consent is per-URL and per-chat-session —
  it does not blanket-authorize future conversions.
- An "I own this site" assertion at confirmation that also registers a per-origin
  robots override for the session.
- Staged progressive-reveal progress notifications during a successful conversion
  (screenshot → theme tokens → digest written) streamed to the builder.
- Producing and persisting a per-site TranscriptionDigest artifact (theme tokens,
  per-page plan, R2-mirrored asset inventory, mirror summary) and returning only a
  completion summary `{ kind, digestKey, summary{ pageCount, assetCount, mirrored,
  mirrorFailures } }` — never a synthesized site.
- Wiring the reproduce-a-website how-to guidance into the builder AI's system
  prompt so the chat AI acts as the synthesizer (reads the digest, then
  reconstructs the draft via its structured-edit and page-CRUD tools).
- End-to-end coverage of the single-page "killer demo" and the multi-page flow.

Out of scope (covered by sibling stories in this capability):
- TranscriptionDigest derivation primitives and the digest read-back action.
- Asset mirroring to R2.
- The page-CRUD structured-edit tools.
- The ConvertConfirmation / TranscribeProgress chat-card variants.

This flow is purely mechanical: there is no internal LLM site-synthesis step and
no fallback path to legacy synthesis behavior (per CLAUDE.md "No Legacy Modes").
The chat AI is the synthesizer.

## Technical Context
Reconciled from BUNDLE-4: REQ-28 (Site Transcription Layer B) **as reshaped by
REQ-30** (mechanical transcribe_site + R2 digest + AI-driven reconstruction).
The matrix reflects the cumulative net end state; REQ-28's internal Opus-synthesis
path was deleted within the same bundle and is NOT documented as a capability.

Architecture: Cloudflare Workers operator API; ASSETS_BUCKET (R2) holds the digest
artifact; FETCH_CACHE_KV holds analyzed Reference Digests.

Implementation deviations from the intent body, surfaced for regression:
- **Digest lookup source.** The intent text says "fetch the digest from chat
  history". REQ-23/REQ-24 (persistent chat) are deferred, so the action reads the
  analyzed digest from the `analyze_page` KV digest cache keyed by
  `sha256(url|schemaVersion)`. The `digestId` input is the source URL the operator
  pasted; conversion of a never-analyzed URL fails with `digest_not_found`.
- **In-memory chat metadata.** Confirmation consent and per-origin robots overrides
  live in a module-singleton store keyed by `(account_id, session_id)`. They survive
  within a Worker isolate's lifetime but NOT across isolate eviction or browser
  reload — an explicit demo trade-off (replaced by a chat-sessions column when
  REQ-23 lands).
- **Synchronous staging.** Stages run sync-await inline within the action rather than
  returning at stage 3 with asset mirroring continuing in a detached background task.
  SSE progress still emits per-stage and per-asset. The emitted stage numbering is an
  artifact of the REQ-28→REQ-30 reshape (screenshot=1, theme=2, asset events=4,
  final digest=3) and does not reflect a 1→2→3→4 ordering.
- **siteId == accountId.** The digest is written to `sites/{accountId}/transcription/
  digest.json`; current code uses the account id as the site id.
- **LLM mocked in tests.** The end-to-end demos drive a scripted (mocked) chat loop;
  real-Claude conversion is a manual smoke test, not a gated test.
- **Per-module confidence dropped** (REQ-30 OUT). Confidence is implicit in the AI
  narrative; only a theme-token confidence band is emitted at the theme stage.
