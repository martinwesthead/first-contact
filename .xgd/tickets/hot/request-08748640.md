## Summary

Updated `docs/llm-context/reproducing-a-website.md` (the LLM system-prompt context for the convert flow) so the assistant no longer pauses after `transcribe_site` returns.

## Motivation

Operator reported having to ask twice to convert a site. Root cause: after `transcribe_site` returned, the assistant narrated next steps and stopped instead of chaining into `read_transcription_digest` and module population. By the time the operator nudged again, the transcription digest had expired and the work had to be re-run.

## Changes

- Added new **Section 1a** ("Execute the entire convert flow in a single turn") to the convert-flow context doc. Explicitly forbids pausing after `transcribe_site` returns, states there are no human-in-the-loop steps, and explains the digest-expiry consequence so future LLM sessions understand *why* pausing breaks the flow.
- Extended **Section 6 (Fallback policy)** to cover the partial-build failure case: report failure immediately, ask the operator to paste the URL again, never leave a half-populated draft un-explained.

## Files touched

- `docs/llm-context/reproducing-a-website.md`

## Notes

Documentation/configuration change only — no code changes, no UATs required per free-coding rules. The doc is loaded into the assistant's system prompt at runtime.
