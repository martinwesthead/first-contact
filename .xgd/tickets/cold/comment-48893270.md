Created REQ-15 — Responsive dials across all dials in the framework.

Key deliverables:
- DOC-7 §3.2 amendment: dial value can be `string | { mobile: V, tablet?: V, desktop?: V }`; `mobile` is the base, others override at their breakpoints, missing levels inherit from below
- Validator: both shapes supported; object form requires `mobile`, rejects unknown breakpoint keys, validates each present value against the dial's enum
- Framework: per-breakpoint class emission (`<dial>-mobile-<v>`, `<dial>-tablet-<v>`, `<dial>-desktop-<v>`); per-site generated CSS gains media-query blocks keyed by token contract breakpoints (sm/md/lg)
- Tool surface: `set_module_dial` accepts the extended value shape; schema description spells out both forms
- All existing modules (hero, photo-text, services-grid, text-block, contact-form) updated via shared `dialClasses(name, value)` helper
- Backward compat: scalar dial values render identically pre- and post-REQ
